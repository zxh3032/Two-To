# Two-To 产品功能规划

> 当前版本聚焦把“认识自己、认识宠物、建立档案、长期照料”这条主路径做扎实；商业化、领养撮合、交易闭环暂不进入第一阶段规划。

## 1. 产品定位

Two-To 面向想养宠物、已经在养宠物、以及正在重新理解自家宠物的人，提供围绕小动物习性、养宠人特征、宠物档案和日常照料的产品能力。

产品不只回答“适合养什么”，也要继续回答：

- 这个物种或品种真实养起来是什么状态。
- 我的生活条件和心理预期是否适合它。
- 已经养了之后，如何更好地理解和照顾它。
- 长期陪伴中，如何记录它的健康、行为、成长和家庭关系。

因此，Two-To 的核心路径可以归纳为：

```text
认识自己 -> 认识宠物 -> 双向适配 -> 建立档案 -> 长期陪伴
```

## 2. 第一版范围调整

### 2.1 第一版目标

第一版不追求平台规模和复杂交易能力，优先做一个清晰、可用、有持续记录价值的宠物产品基础版本。

第一版目标：

- 让想养宠用户能了解不同小动物的真实习性和照护成本。
- 让用户通过测评获得相对合理的养宠适配分析。
- 让已养宠用户可以建立宠物档案，并记录基础照料信息。
- 通过知识库、测评结果、档案记录形成后续功能的基础数据。

### 2.2 第一版暂不做

- 不做领养机构后台。
- 不做宠物交易、订单、支付、佣金、履约等商业化闭环。
- 不做复杂社区运营体系。
- 不做医疗诊断或替代兽医判断的结论。
- 不做重度 AI 自动决策，第一版更适合以结构化规则、解释性推荐和人工可维护内容为主。

### 2.3 第一版可以保留的远期入口

后续如果要支持宠物寻找家庭，可以先考虑轻量的信息发布和自主联系能力：

- 用户或机构发布宠物基础信息。
- 平台提供标签、适配提醒和基础风险提示。
- 联系、沟通、线下确认由双方自行完成。

这部分暂不进入第一版，只作为未来扩展方向保留。

## 3. 第一版核心功能

### 3.1 物种与品种知识库

知识库是 Two-To 的基础能力。它不应只做百科介绍，而应围绕真实养宠体验组织内容。

核心内容：

- 物种：猫、狗、兔、仓鼠、鸟类、龟类、鱼类、蜥蜴、蛇、昆虫等。
- 品种或类型：例如英短、布偶、边牧、柯基、垂耳兔、金丝熊、玄凤鹦鹉等。
- 习性特征：活动时间、社交需求、独处能力、攻击性、胆小程度、叫声、体味、掉毛、破坏性。
- 照护难度：饮食、清洁、运动、训练、环境布置、温湿度要求。
- 健康风险：常见疾病、遗传病倾向、体重风险、寿命范围。
- 成本估算：首月准备成本、月均消耗、医疗风险成本。
- 适配标签：适合新手、适合小户型、适合高陪伴、适合安静家庭、不适合频繁出差等。

首期建议先覆盖猫狗常见品种，同时预留多物种字段，不要把模型设计死在猫狗上。

### 3.2 养宠人特性测评

测评用于帮助用户认识自己的生活状态，而不是制造绝对答案。

测评维度：

- 居住条件：面积、是否租房、是否允许宠物、是否有阳台或院子。
- 时间精力：工作时长、出差频率、每日陪伴时间、周末活动规律。
- 预算水平：月预算、医疗应急预算、用品接受度。
- 家庭成员：儿童、老人、过敏人群、已有宠物。
- 性格偏好：喜欢安静、互动、训练、陪伴、观赏，还是低打扰。
- 接受边界：掉毛、叫声、气味、清洁频率、家具损耗。
- 养宠经验：新手、养过同类宠物、能否处理疾病和行为问题。

输出结果：

- 养宠人画像。
- 适合的物种或品种方向。
- 不建议选择的类型。
- 需要提前补齐的条件。
- 养宠前准备清单。

### 3.3 宠物适配结果

适配结果不要只给排行榜，应强调解释和风险提示。

推荐结果结构：

- 推荐物种或品种。
- 匹配分数或匹配等级。
- 匹配原因。
- 潜在冲突点。
- 养前准备建议。
- 替代选择。

示例：

- “你适合低运动量、低噪音、可独处能力较强的宠物。”
- “不建议优先考虑高运动量犬种，因为你的日常陪伴时间和运动安排不足。”
- “如果仍然喜欢这类宠物，需要提前准备固定运动时间、训练计划和备用照看方案。”

### 3.4 宠物档案

宠物档案服务于已养宠用户，是长期留存的核心。

基础信息：

- 名字、头像、物种、品种、性别、生日、到家日。
- 体重、绝育状态、芯片号、疫苗状态。
- 性格标签、喜好、害怕的事物、饮食偏好。

记录信息：

- 健康记录：疫苗、驱虫、体检、用药、就诊、过敏、病史。
- 成长记录：照片、事件、体重变化、行为变化。
- 日常记录：喂食、饮水、排便、清洁、洗澡、剪甲。
- 重要提醒：下一次疫苗、驱虫、复诊、用品补货。

### 3.5 日常照料工具

第一版可以做轻量工具，避免一开始做成复杂管理系统。

建议优先级：

- 疫苗和驱虫提醒。
- 体重记录。
- 喂食记录。
- 就诊和用药记录。
- 宠物开销记录。
- 用品清单和补货提醒。

这些工具能够让产品从“养前决策”自然延伸到“养后陪伴”。

### 3.6 成长记录

成长记录强化情感价值，也能补充宠物个体习性。

功能方向：

- 时间线记录。
- 照片和文字日记。
- 重要节点：第一次到家、第一次体检、生日、学会新技能、生病康复。
- 行为观察：亲人程度、胆量变化、食欲变化、睡眠变化。

## 4. 后续功能规划

### 4.1 行为观察与习性分析

围绕宠物行为建立记录和解释能力。

- 异常行为记录：乱叫、乱尿、拆家、攻击、躲藏、挑食、频繁舔毛。
- 触发因素记录：搬家、新成员、换粮、天气、发情、疾病、环境噪音。
- 行为趋势：某个行为出现频率是否增加。
- 处理建议：先给常识性排查，不做医疗诊断。

这个方向能让用户更理解“自家的这一只”，不是只理解品种平均值。

### 4.2 健康管理增强

在第一版基础健康记录之上，逐步扩展为完整健康档案。

- 体重曲线和异常波动提醒。
- 疫苗、驱虫、体检计划。
- 用药周期和服药提醒。
- 就诊记录、检查报告、处方照片归档。
- 慢病管理，例如皮肤病、肠胃敏感、泌尿问题。
- 老年宠物照护计划。

### 4.3 饮食与营养管理

不同小动物的饮食差异很大，适合成为长期特色能力。

- 主粮、罐头、冻干、零食记录。
- 每日喂食量建议和实际记录。
- 换粮计划。
- 过敏源记录。
- 饮水量观察。
- 特殊物种饮食清单，例如兔草、鸟粮、水族饲料、爬宠活体饲料。

### 4.4 环境与栖息地管理

如果产品希望覆盖更多小动物，就不能只围绕猫狗设计。

- 笼舍、猫砂盆、鱼缸、龟缸、爬箱、鸟笼等环境档案。
- 温度、湿度、光照、过滤、垫材、躲避屋等配置记录。
- 清洁周期提醒。
- 设备维护提醒，例如过滤器、加热灯、UVB 灯。
- 环境变化和宠物状态关联记录。

这会让 Two-To 在多物种方向上更有辨识度。

### 4.5 训练与互动计划

面向狗、猫、鸟类等有互动训练需求的宠物。

- 基础训练计划。
- 每日训练打卡。
- 训练目标和阶段进度。
- 常见问题拆解。
- 宠物互动游戏推荐。

### 4.6 家庭协作

很多宠物不是一个人在养，需要多人协作。

- 家庭成员加入宠物档案。
- 分配喂食、清洁、遛狗、复诊等任务。
- 共享提醒。
- 记录责任人。
- 多宠家庭管理。

### 4.7 开销与预算

帮助用户真实了解养宠成本。

- 按宠物记录开销。
- 分类统计：食物、用品、医疗、美容、寄养、训练。
- 月度趋势。
- 养前预算估算和养后真实开销对比。
- 高风险医疗储备提醒。

### 4.8 宠物安全与应急

这是高实用价值方向，不涉及商业化。

- 宠物应急卡：姓名、照片、病史、联系人、常去医院。
- 走失信息卡。
- 临时照看说明。
- 紧急就医清单。
- 危险食物和危险植物查询。

### 4.9 内容与经验沉淀

可以先做结构化内容，而不是复杂社区。

- 新手专题。
- 物种照护手册。
- 品种避坑指南。
- 用户自己的养宠笔记。
- 知识内容收藏。
- 基于宠物档案推荐相关内容。

### 4.10 轻量信息发布

远期可以提供非商业化、非闭环的平台能力。

- 宠物信息发布。
- 自主联系入口。
- 基础标签筛选。
- 风险提示和责任声明。
- 平台不介入交易、支付、履约。

这部分应放在核心工具和档案体系成熟之后再考虑。

## 5. 版本节奏建议

### V0.1 基础骨架

目标：完成可浏览、可测评、可建档的最小产品闭环。

- 首页功能入口。
- 品种知识库列表和详情。
- 适配测评问卷。
- 测评结果页。
- 宠物档案基础信息。
- 健康提醒基础字段。

### V0.2 记录闭环

目标：让已养宠用户愿意持续使用。

- 成长时间线。
- 疫苗、驱虫、体重、就诊记录。
- 喂食和清洁提醒。
- 宠物开销记录。
- 多宠档案。

### V0.3 个体理解

目标：从“品种平均值”走向“理解自家宠物”。

- 行为观察记录。
- 习性标签。
- 异常趋势提醒。
- 饮食偏好和过敏记录。
- 个体档案分析。

### V0.4 多物种增强

目标：突破猫狗工具，覆盖更多小动物。

- 物种差异化字段。
- 栖息地和环境管理。
- 特殊饮食记录。
- 多物种知识库专题。

### V1.0 完整陪伴体验

目标：形成从养前决策到养后陪伴的完整产品体验。

- 适配测评成熟版。
- 品种知识库成熟版。
- 宠物档案成熟版。
- 健康、饮食、行为、开销、成长记录闭环。
- 家庭协作。
- 应急卡和临时照看说明。

## 6. 产品原则

- 先做真实有用的功能，再做平台规模。
- 推荐结果必须可解释，不制造绝对答案。
- 多物种优先作为底层设计原则，避免只服务猫狗。
- 健康相关内容只做记录、提醒和常识性建议，不替代兽医。
- 商业化、交易、领养撮合暂缓，避免第一版目标失焦。
- 情感记录和理性管理并重，让产品既有工具价值，也有陪伴感。
