# 本地开发环境说明

本文档记录 Two-To 当前前后端初始化后的本地启动、联调和校验方式。

## 环境要求

- Go：当前仓库使用 `1.24.4`。
- Node.js：建议使用 `20.x` 或更高的 LTS 版本。
- npm：前端当前使用 npm 管理依赖。
- MySQL：后端已接入 GORM + MySQL，初始化阶段允许不配置 DSN，服务会跳过数据库连接。

## 后端启动

后端默认监听 `:8080`，配置来自环境变量。

```bash
cp backend/.env.example backend/.env
cd backend
go run .
```

也可以从根目录执行：

```bash
make backend-dev
```

常用环境变量：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `TWO_TO_APP_NAME` | `two-to-api` | 服务名，会出现在日志和 ping 响应中 |
| `TWO_TO_ENV` | `development` | 运行环境 |
| `TWO_TO_HTTP_ADDR` | `:8080` | HTTP 监听地址 |
| `TWO_TO_LOG_LEVEL` | `debug` | zap 日志级别 |
| `TWO_TO_MYSQL_DSN` | 空 | MySQL DSN，为空时跳过数据库初始化 |

验证接口：

```bash
curl http://localhost:8080/health
curl http://localhost:8080/api/v1/ping
curl http://localhost:8080/api/v1/slogan
```

## 前端启动

前端默认使用 Vite dev server，监听 `5173` 端口。

```bash
cp frontend/.env.example frontend/.env
cd frontend
npm install
npm run dev
```

也可以从根目录执行：

```bash
make frontend-install
make frontend-dev
```

前端通过 `VITE_API_BASE_URL` 访问后端，默认值是：

```text
http://localhost:8080
```

## 常用校验

根目录提供了统一命令：

```bash
make backend-test-linux
make frontend-lint
make frontend-test
make frontend-build
make check
```

后端修改 proto 后需要重新生成 Go 代码：

```bash
make backend-proto
```

## 当前联调链路

初始化阶段已经打通最小链路：

```text
Vite React 页面
  -> shared/api/system.fetchSlogan
    -> GET /api/v1/slogan
      -> Gin router
        -> controllers/slogan
          -> models/page/slogan
            -> proto.SloganResponse
  -> shared/api/system.fetchPing
    -> GET /api/v1/ping
      -> Gin router
        -> controllers/ping
          -> models/page/ping
            -> proto.PingResponse
```

首页控制台会展示 `/api/v1/slogan` 的项目口号和 `/api/v1/ping` 的联通状态。
