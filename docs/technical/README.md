## 技术原则

- Web 端优先，先保证核心体验闭环，再扩展小程序或 App。
- 前后端在一个 monorepo 中统一管理，方便早期联动开发。
- 请求参数和返回值使用 proto 统一定义，再分别生成前端、后端类型。
- 文档和代码同步提交，重要目录、分层、接口约定都需要在文档中留下记录。
